4 minutes de lecture

Le modèle OSI et le protocole TCP/IP

Le protocole TCP/IP (Transmission Control Protocol / Internet Protocol) est basé sur le modèle OSI (Open Systems Interconnection).

le modèle OSI a été conçu dans les années 1970 au sein de l’ISO (International Organization for Standardization). le modèle permet une communication standard en réseau de tous les systèmes informatiques. Il permet de décrire couche par couche les fonctionnalités nécéssaires à la communication.

Définition des couches

  • La couche 1
    • Nom : Physique.
    • Rôle : Permet d’ofrrir un support de transmission pour la communication.
    • Matériel associé : Le hub (ou concentrateur en français).
  • La couche 2
    • Nom : Liaison de données.
    • Rôle : Connecter les machines entre elles sur un réseau local.
    • Matériel associé : Switch, hub.
  • La couche 3
    • Nom : Réseau.
    • Rôle : Interconnecte les réseaux entre eux.
    • Matériel associé : Le routeur.
  • La couche 4
    • Nom : Transport.
    • Rôle : Gère les connnexions applicative.
    • Matériel associé : /
  • La couche 5
    • Nom : Session.
    • Rôle : Gére la synchronisation des échanges et les « transactions ». Elle gère aussi l’ouverture et la fermeture de session.
    • Matériel associé : /
  • La couche 6
    • Nom : Présentation.
    • Rôle : Elle est chargée du codage des données applicatives, précisément de la conversion entre données manipulées au niveau applicatif et chaînes d’octets effectivement transmises.
    • Matériel associé : /
  • La couche 7
    • Nom : Présentation
    • Rôle : /
    • Matériel associé : Le proxy

⚠️ Le modèle OSI est un modèle théorique sur lequel le modèle TCP/IP se repose. Dans son implémentation, il n’utilise pas les couches 5 et 6. Dans la pratique, le procole TCP/IP n’utilise donc que 5 couches.

Modèle par architecure

Couches Hautes Données 7 Application Point d'accès aux services réseau.
6 Présentation Gère le chiffrement et le déchiffrement des données, convertit les données machine en données exploitables par d'autre machine.
5 Session Communication Interhost, gère les sessions entre les différentes applications
Segment 4 Transport Connexion de bout en bout, connectabilité, contrôle des flux et notion de port (protocole TCP et UDP)
Couches matérielles Paquet 3 Réseau Détermine le parcours des données et l'adressage logique (IP)
Trame 2 Liaison de données Adressage physique
Bit 1 Physique Transmission des signaux sous forme numérique ou analogique

Les rêgles d’or du modèle OSI

Le modèle OSI étant une norme, il est nécessaire d’indiquer correctement comment travailler pour mettre en place des réseaux. Cela permet de garantir une compabilité.

Chaque couche à un rôle spécifique, aucun autre couche s’occupera de faire le travail d’une autre.

Chaque couche est indépendante.

Quand un ouvre un navigateur et qu’on ouvre une page internet, on utilise sans le savoir le modèle OSI. La couche 7 application (le navigateur) s’adresse aux couches réseau pour traiter la demande de l’utilisateur.

Chaque couche ne peut communiquer qu’avec une couche adjacente.

Quand on envoie des données sur le réseau, on va de la couche 7 à la couche 1 :

application => présentation => session => transport => réseau => liaison de données => physique

Quand on reçois des données provenant du réseau, on va de la couche 1 à la couche 7 :

physique => liaison de données => réseau => transport => session => présenation => application.

En résumé

Le protocole par vulgarisation : https://www.youtube.com/watch?v=26jazyc7VNk

  • Le modèle OSI est une norme précisant comment les machines doivent communiquer entre elles.
  • C’est un modèle théorique, celui réellement utilisé est le protocole TCP/IP.
  • Il possède 7 couches.
  • Chaque couche à un rôle spécifique.
  • Les couches 1 à 4 sont les couches réseaux. Elles offrent le servie de communication à la couche applicative.
  • Chaque couche est indépendante des autres.
  • Les couches ne peuvent communiquer qu’avec les couches adjacentes.
  • Quand on envoie des données, on va de la couche 7 à 1 et inversement quand on en reçois.
